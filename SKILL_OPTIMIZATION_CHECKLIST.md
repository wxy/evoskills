# Skill Optimization Checklist

> 技能优化系统计划 - 为 v2.0.0 统一和优化所有技能定义

---

## 优化标准

### 目标行数范围

- **Tier 1 (核心技能)**: 100-200 行（可更长，需要详细解释）
- **Tier 2 (系统技能)**: 80-150 行（需要清晰明确）
- **Tier 3 (可选技能)**: 60-120 行（需要精炼）

### 优化范围

1. **移除过时内容**
   - 删除 v1.x 的步骤和示例
   - 删除已弃用的工作流程
   - 删除冗余的说明

2. **整合重复内容**
   - 合并相似的步骤
   - 消除重复的解释
   - 统一术语

3. **现代化表达**
   - 更新为 v2.0.0 架构
   - 使用统一的英文表达
   - 添加清晰的结构标题

4. **保留关键内容**
   - 当前工作流程
   - 实际使用示例
   - 依赖关系声明

## 优化流程

### 对于每个技能：

1. **备份当前版本**
   ```bash
   # 在修改前记录原始行数
   wc -l .agent/skills/_X/SKILL.md
   ```

2. **分析内容**
   - 读取完整文件
   - 识别每个部分的目的
   - 标记可删除的内容

3. **重构文件**
   - 删除 v1.x 的历史内容
   - 整合重复的部分
   - 保留当前工作流程
   - 更新示例和引用

4. **验证内容**
   - 确保所有关键功能仍有说明
   - 验证示例仍然有效
   - 检查依赖关系声明

5. **提交更改**
   ```bash
   # 格式: perf(skill-name): Optimize SKILL.md, reduce bloat
   git commit -m "perf(_skill-name): Optimize SKILL.md ${old_lines}→${new_lines} lines"
   ```
